# Product

## Register

product

## Users

Developers and power users of graph-memory. Two modes of use, equally important:

1. **Glance mode.** Mid-coding session, quickly verifying memory state, checking pipeline health, scanning the latest brief. In and out in seconds.
2. **Deep inspection mode.** Debugging graph topology, auditing delta chains, tracing pipeline failures, reading dream fragments. Sustained focus, lots of scrolling and cross-referencing.

They work at a desk, in a code editor, often in a terminal. The dashboard is a secondary surface, open alongside their primary work.

## Product Purpose

An operator surface for runtime health, startup context, pipeline jobs, and graph memory. It exposes the internals of a persistent knowledge graph so a developer can inspect, debug, and monitor what their agent remembers across sessions.

Success means: the user can answer "what does my agent know right now?" and "is the memory pipeline healthy?" without reading log files.

## Brand Personality

Precise. Calm. Technical.

Understated competence. The interface earns trust by being right, not by being flashy. Surgical clarity over visual noise. Every pixel works; nothing is decorative for its own sake.

Terminal-native aesthetic: monospace-forward, minimal chrome, data-first. It should feel like a well-configured development environment, not a marketing page.

## Anti-references

Generic SaaS dashboards. The standard sidebar + card grid + metric hero template. Anything that looks like it was assembled from a component library without thinking.

Overly decorative approaches: glassmorphism, gratuitous gradients, animated transitions that slow down scanning.

"Modern" used as a synonym for rounded corners and purple accents.

## Design Principles

1. **Data density over decoration.** Show more information, not more chrome. White space is structural, not aesthetic filler.
2. **Terminal confidence.** The interface should feel like a well-configured terminal: precise, fast, predictable. No surprises in layout or interaction.
3. **Depth on demand.** A casual glance gives status. A click reveals layers of detail. The surface is calm; the depth is there when needed.
4. **Monospace as identity.** The typeface is the personality, not a concession. Hierarchy through weight and spacing, not font family switches.

## Accessibility & Inclusion

WCAG 2.1 AA. Standard contrast ratios, keyboard navigation, focus management. Reduced motion respected. No special requirements beyond this baseline.
